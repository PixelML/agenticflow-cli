       ```ts
       if (opts.platform) {
         try {
           const skills = await fetchPlatformSkills({ token: process.env.GITHUB_TOKEN });
           const installedNames = new Set<string>(
             listInstalledPacks().flatMap((p) => p.skill_names ?? [])
           );
           const limited = typeof opts.limit === 'number' && opts.limit > 0
             ? skills.slice(0, opts.limit)
             : skills;
           const items = limited.map((s) => ({
             name: s.name,
             description: s.description,
             pack: s.pack,
             installed: installedNames.has(s.name),
           }));
           if (opts.json) {
             printJson({
               schema: 'agenticflow.platform.skill.list.v1',
               count: items.length,
               platform: true,
               items,
             });
             return;
           }
           // Human output per D-05
           for (const it of items) {
             const prefix = it.installed ? '✓ ' : '  ';
             const tag = it.installed ? `(${it.pack})` : '(platform)';
             console.log(`${prefix}${it.name}  ${tag}  ${it.description}`);
           }
           console.log(`\n${items.length} platform skill${items.length === 1 ? '' : 's'}`);
           return;
         } catch (err) {
           if (err instanceof PlatformCatalogError) {
             fail({ code: err.code, message: err.message, hint: err.hint });
             return;
           }
           throw err;
         }
       }
       // ... existing no-flag logic continues UNCHANGED below
       ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06 | Tampering | --limit user input | mitigate | `parseInt(v, 10)` and `opts.limit > 0` guard before slicing — negative or NaN values fall through to "no cap" branch, never throw |
| T-05-07 | Information Disclosure | GITHUB_TOKEN in error output | mitigate | Token only flows from `process.env.GITHUB_TOKEN` into platform-catalog.ts opts; never logged in main.ts. PlatformCatalogError messages from Plan 01 are token-free by contract |
| T-05-08 | Denial of Service | Unbounded skill list rendering | accept | Output is plain text to stdout, GitHub repo bounded to ~18 packs × ~5 skills. Memory cost negligible |
| T-05-09 | Spoofing | Installed-checkmark false positive | accept | Skill name collisions across packs are a known UX tradeoff per D-04. Consequence is cosmetic only — no security impact |
</threat_model>
