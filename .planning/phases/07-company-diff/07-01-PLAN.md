## Goal

Add `af company diff <file>` — a read-only command that compares a local YAML company export against the live workspace and reports field-level differences. Reuses existing `exportCompany()` and `changedFields()`. No new SDK calls, no writes.

## Context

Phase 7 — see `.planning/phases/07-company-diff/07-CONTEXT.md` for locked decisions D-01 through D-07.

Reuses existing primitives in `packages/cli/src/cli/company-io.ts`:
- `exportCompany(client, cliVersion)` — fetches live workspace state as `CompanyExportSchema`
- `changedFields(exported, existing)` — JSON.stringify field-level comparison, returns `string[]`
- `parseYaml`, `CompanyExportSchema`, `COMPANY_EXPORT_FIELDS`

CLI wiring follows existing `companyCmd.command("export"|"import")` pattern in `main.ts` (~lines 5344-5461).

export type DiffAgentStatus = "new" | "modified" | "remote_only" | "in_sync";

export interface DiffAgentEntry {
  name: string;
  status: DiffAgentStatus;
  changed_fields: string[]; // empty for new/remote_only/in_sync
}

export interface CompanyDiffResult {
  schema: "agenticflow.company.diff.v1";
  in_sync: boolean;
  summary: { new: number; modified: number; remote_only: number; in_sync: number };
  agents: DiffAgentEntry[];
}

2. Function:
```ts
export async function diffCompany(
  client: AgenticFlowClient,
  localSchema: CompanyExportSchema,
): Promise<CompanyDiffResult>
```

Implementation:
- Validate `localSchema.schema === "agenticflow.company.export.v1"`; if not, throw `CompanyIOError("Unsupported schema version: ...", "schema_version_mismatch")` (mirror importCompany guard exactly).
- Call `client.agents.list({ projectId, limit: 1000 })` and use `extractAgentsFromListResponse` (export this helper if not already exported — add `export` keyword to its declaration).
- Build `existingByName: Map<string, Record<string, unknown>>` keyed by agent `name`.
- Build `localByName: Map<string, CompanyExportAgentEntry>` keyed by `name`.
- For each local entry: if name in existingByName → use `changedFields(entry, existing)`; if non-empty → status `"modified"` with `changed_fields`; if empty → status `"in_sync"` with `[]`. If not in existingByName → status `"new"` with `[]`.
- For each existing agent whose `name` is NOT in localByName → status `"remote_only"` with `[]`.
- Compute `summary` counts. `in_sync = summary.new === 0 && summary.modified === 0 && summary.remote_only === 0`.
- Sort `agents` by name ascending for deterministic output.
- Return `{ schema: "agenticflow.company.diff.v1", in_sync, summary, agents }`.

   - `diffCompany marks differing agents as modified with changed_fields list`
   - `diffCompany throws schema_version_mismatch on bad schema`
   - `diffCompany sorts agents by name`

<action>
Add a new `companyCmd.command("diff")` subcommand in `packages/cli/src/cli/main.ts` immediately after the existing `companyCmd.command("import")` block. Match the surrounding code style exactly (Commander chain, async action handler, try/catch, fail() helper, printResult helper).

Behavior:

1. Signature: `.argument("<file>", "Path to local company YAML export")` and `.option("--json", "Output machine-readable JSON")`.
2. Description: `"Diff a local company YAML export against the live workspace"`.
3. Help text MUST include the line: `"Exit codes: 0 = in sync, 1 = differences found"`.
4. Action handler steps:
   a. Resolve and validate file path: `const filePath = resolve(file);`. If `!existsSync(filePath)` → call `fail("file_not_found", \`Company file not found: ${filePath}\`, { hint: "Check the path and try again" })`.
   b. Read file: `const raw = readFileSync(filePath, "utf8");`. Parse with `parseYaml(raw)` inside a try/catch — on error throw via `fail("invalid_yaml", \`Failed to parse YAML: ${err.message}\`, { hint: "Verify file is valid YAML produced by 'af company export'" })`.
   c. Build authenticated client (use the same pattern as `companyCmd.command("import")` — `getClientOrFail()` or whatever helper that block uses).
   d. Call `const result = await diffCompany(client, parsed as CompanyExportSchema);`. Wrap in try/catch — on `CompanyIOError` → `fail(err.code, err.message)`.
   e. If `opts.json`:
      - `printResult("agenticflow.company.diff.v1", result);` (or whatever the existing printResult helper signature is — match how `import --json` does it).
   f. Else (human-readable):
      - For each agent in `result.agents`:
        - `new` → `console.log(\`+ ${name}\`)`
        - `modified` → `console.log(\`~ ${name} (fields: ${changed_fields.join(", ")})\`)`
        - `remote_only` → `console.log(\`< ${name}\`)`
        - `in_sync` → skip (do not print)
      - If `result.in_sync` → `console.log("✓ In sync — no differences found")`.
   g. Exit code: if `result.in_sync` → `process.exit(0)`; else → `process.exit(1)`. Apply this AFTER both human and JSON branches (single exit at end of handler).

Imports to add at top of main.ts if not already present: `diffCompany`, `CompanyExportSchema` from `./company-io.js`.

Do NOT modify any other command. Do NOT change exit code behavior of existing commands.
</action>

## Threat Model

Trust boundary: untrusted local file path + untrusted YAML content crossing into the CLI process.

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-07-01 | Tampering / Input Validation | `<file>` argument in main.ts diff handler | mitigate | `resolve()` + `existsSync()` check before `readFileSync`; structured `file_not_found` error with hint, no stack trace |
| T-07-02 | Tampering / Injection | YAML parsing in diff handler | mitigate | `parseYaml` wrapped in try/catch; malformed input → `fail("invalid_yaml", ...)` with hint, never bubbles raw parser stack to user |
| T-07-03 | DoS / Resource Exhaustion | Oversized YAML file | accept | CLI is local single-user tool; user controls input; no network exposure. Documented as accepted risk. |
| T-07-04 | Information Disclosure | Error messages leaking absolute paths | accept | CLI is local; absolute paths in errors are expected/desired for debugging. No multi-tenant context. |
| T-07-05 | Tampering / Schema confusion | YAML claims wrong schema version | mitigate | `diffCompany` validates `schema === "agenticflow.company.export.v1"` and throws `CompanyIOError("schema_version_mismatch")`, mirroring importCompany guard |
| T-07-06 | Repudiation | Diff results affect downstream pipelines | mitigate | Deterministic alphabetical sort of agents array; stable JSON.stringify field comparison; documented exit code contract (0 = sync, 1 = differs) |
